Want to make your LLM inference 2x faster? It's easier than you think. This guide walks you through setting up momo-kibidango in under 5 minutes.

## Prerequisites: What You Need

Before we start, make sure you have:

- **Mac with Apple Silicon** (M1/M2/M3/M4)
- **macOS 13.0+** (Ventura or newer)
- **Python 3.9+** (check with `python3 --version`)
- **16GB RAM** recommended (works with 8GB in reduced mode)
- **50GB free disk space** for models
- **Internet connection** for model downloads

Not sure about your system? Run this check:

```bash
# System check script
curl -fsSL https://raw.githubusercontent.com/rdreilly58/momo-kibidango/main/check_system.sh | bash
```

## Installation: Three Simple Commands

### Step 1: Clone the Repository

```bash
git clone https://github.com/rdreilly58/momo-kibidango.git
cd momo-kibidango
```

### Step 2: Set Up Python Environment

```bash
# Create virtual environment (recommended)
python3 -m venv venv
source venv/bin/activate

# Install dependencies
pip install -r requirements.txt
```

### Step 3: Download Models

```bash
# Automated download (uses Hugging Face)
python setup.py --download-models

# This downloads:
# - Draft model (1B params, ~2GB)
# - Qualifier model (3B params, ~6GB)  
# - Target model (7B params, ~14GB)
# Total: ~22GB download, ~25GB on disk
```

**Expected output:**
```
Downloading draft model... ✓ (2.1GB)
Downloading qualifier model... ✓ (6.3GB)
Downloading target model... ✓ (14.2GB)
Optimizing for Apple Silicon... ✓
Models ready! Total time: 4m 32s
```

## Basic Usage Examples

### Example 1: Simple Q&A

```python
from momo_kibidango import MomoInference

momo = MomoInference()

# Quick questions get the best speedup
questions = [
    "What's the capital of Japan?",
    "How do you say 'hello' in Spanish?",
    "What's 15% of 200?"
]

for q in questions:
    response = momo.generate(q, max_tokens=50)
    print(f"Q: {q}")
    print(f"A: {response}\n")
```

### Example 2: Code Generation

```python
# Code generation with syntax highlighting
prompt = """Write a Python function that finds the nth Fibonacci number.
Include docstring and type hints."""

response = momo.generate(
    prompt, 
    max_tokens=200,
    temperature=0.3  # Lower temp for code
)

print("Generated code:")
print(response)
```

### Example 3: Creative Writing

```python
# Creative tasks with higher temperature
prompt = "Write a haiku about machine learning"

response = momo.generate(
    prompt,
    max_tokens=50,
    temperature=0.9,  # Higher for creativity
    top_p=0.95
)

print(response)
```

### Example 4: Batch Processing

```python
# Process multiple prompts efficiently
prompts = [
    "Summarize the benefits of exercise",
    "List 5 Python best practices",
    "Explain quantum computing simply"
]

responses = momo.generate_batch(
    prompts,
    max_tokens=150,
    batch_size=8  # Optimal for memory
)

for prompt, response in zip(prompts, responses):
    print(f"Prompt: {prompt}")
    print(f"Response: {response}\n")
```

## Configuration Options

### Basic Configuration

```python
from momo_kibidango import MomoInference, Config

config = Config(
    # Model selection
    target_model="mistralai/Mistral-7B-Instruct-v0.2",
    draft_model="deepseek-ai/deepseek-coder-1b-base",
    qualifier_model="microsoft/phi-2",
    
    # Performance tuning
    enable_pyramid=True,      # 3-model mode (default)
    max_draft_tokens=8,       # Tokens to speculate
    acceptance_threshold=0.1,  # Stricter = better quality
    
    # Memory management
    max_memory_gb=12,         # Memory limit
    enable_quantization=True,  # 4-bit for draft model
    share_embeddings=True,     # Share across models
    
    # Fallback behavior
    fallback_on_error=True,   # Graceful degradation
    min_speedup_threshold=1.5  # Disable if slower
)

momo = MomoInference(config)
```

### Advanced Tuning

```python
# For specific use cases
config = Config(
    # Code-optimized settings
    temperature_offset=-0.2,   # More deterministic
    repetition_penalty=1.1,    # Avoid loops
    
    # Memory-constrained (8GB Macs)
    enable_pyramid=False,      # 2-model only
    aggressive_quantization=True,
    
    # Quality-first mode
    acceptance_threshold=0.05,  # Very strict
    verify_all_tokens=True,     # No shortcuts
)
```

## Troubleshooting Common Issues

### Issue 1: "Out of Memory" Error

**Symptom:** Process killed or memory warnings

**Solutions:**
```bash
# Reduce memory usage
python setup.py --quantize-models --max-memory 8

# Or use 2-model mode
export MOMO_ENABLE_PYRAMID=false
```

### Issue 2: Slow First Inference

**Symptom:** First generation takes 2-3 minutes

**Solution:** Models loading. Subsequent runs will be fast. Pre-warm:
```python
# Warm up models
momo = MomoInference()
momo.warmup()  # Loads all models
```

### Issue 3: Quality Degradation

**Symptom:** Outputs seem lower quality

**Solution:** Tighten acceptance threshold:
```python
config = Config(acceptance_threshold=0.05)  # Stricter
```

### Issue 4: Import Errors

**Symptom:** `ModuleNotFoundError`

**Solution:**
```bash
# Ensure virtual environment is active
source venv/bin/activate

# Reinstall dependencies
pip install -r requirements.txt --force-reinstall
```

### Issue 5: Model Download Fails

**Symptom:** Connection timeouts or auth errors

**Solution:**
```bash
# Manual download with resume
python setup.py --download-models --resume

# Or use Hugging Face CLI
huggingface-cli login  # If private models
```

## Feature Flags

Control behavior with environment variables:

```bash
# Disable pyramid (use 2-model only)
export MOMO_ENABLE_PYRAMID=false

# Force CPU-only mode (debugging)
export MOMO_DEVICE=cpu

# Custom model paths
export MOMO_MODEL_PATH=/path/to/models

# Disable telemetry
export MOMO_TELEMETRY=false
```
